Critical Settings API Missing Role Check

/api/settings POST endpoint checks for authenticated user but does NOT verify role. Any authenticated user (including contributors) can toggle site settings like inauguration_mode.

Fix: Add role check for super_editor on POST endpoint, matching the pattern used in /api/gallery.

High Gallery Delete is Client-Side

Gallery item deletion happens directly from the browser using supabase.storage.from("media").remove(). URL manipulation could allow deleting arbitrary files from storage.

Fix: Move delete operation to a DELETE /api/gallery endpoint with server-side auth and path validation.

Medium No Input Validation for Gallery Categories

Gallery upload accepts any string for category and aspect_ratio without server-side enum validation. Invalid values can be stored in the database.

Fix: Validate against the CATEGORIES constant array and ['portrait','landscape','square'] before insert.

5. Issues Found & Resolutions

20 Issues

Critical Settings API: Any Authenticated User Can Change Site Settings

The /api/settings POST endpoint only checks for a valid session but doesn't verify the user's role. Contributors can toggle inauguration mode and other settings.

Fix: Add profile role check: if (profile.role !== 'super_editor') return 403

High Splash Screen Shows on Every Visit

The splash screen renders on every homepage load with no session memory. Users must click through it each time, creating friction.

Fix: Use sessionStorage or localStorage to track if user has seen the splash this session. Skip it on subsequent visits.

High No Login Rate Limiting

The editorial login page has no rate limiting, allowing unlimited password attempts. This makes brute-force attacks possible.

Fix: Add IP-based rate limiting (5 attempts/min) to the login form or create a /api/auth/login endpoint with rate limiting.

High Client-Side Gallery Deletion (Path Traversal Risk)

Gallery deletion extracts storage path from URL client-side and calls supabase.storage.remove() directly. Malicious URL manipulation could delete arbitrary files.

Fix: Create a DELETE /api/gallery server endpoint that validates the item belongs to gallery_items table before deleting from storage.

Medium Article Pages Use Client-Side Rendering

Article detail pages use createBrowserSupabaseClient() for data fetching, meaning content loads after JavaScript hydration. This hurts SEO and initial load time.

Fix: Convert to server-side rendering with createServerSupabaseClient() and use generateStaticParams for pre-rendering popular articles.

Medium No Accessibility: prefers-reduced-motion

Multiple simultaneous animations (hero-drift, particles, curtain-wave, pulse-live) run continuously. Users who prefer reduced motion have no option to disable them.

Fix: Add @media (prefers-reduced-motion: reduce) CSS to disable/simplify animations. Add Framer Motion's useReducedMotion() hook.

Medium Missing SEO: Open Graph & Twitter Cards

No Open Graph or Twitter Card metadata configured. Shared links on social media won't show preview images or descriptions.

Fix: Add openGraph and twitter metadata in app/layout.tsx and per-article dynamic metadata in articles/[slug]/page.tsx.

Low No Image Optimization Configuration

Uses raw <img> tags instead of Next.js <Image> component. No remotePatterns configured in next.config for Supabase images.

Fix: Add Supabase domain to next.config.ts remotePatterns and migrate key images to next/image for automatic optimization.

Low No Testing Framework

No test runner (Jest/Vitest), no E2E tests (Playwright/Cypress), no pre-commit hooks. Changes are deployed without automated verification.

Fix: Add Vitest for unit tests, Playwright for E2E tests, and Husky for pre-commit hooks.

Medium SecurityShield Breaks Accessibility

The SecurityShield.tsx component blocks right-click, copy/paste, and keyboard shortcuts globally. This breaks assistive technology users who rely on these features and may violate WCAG accessibility guidelines.

Fix: Remove or significantly tone down SecurityShield. Content protection via JS is easily bypassed anyway. Focus on watermarking images instead.

Medium Related Articles Query Not Category-Filtered

The article detail page's "Related Articles" section fetches ANY 3 published articles instead of filtering by the same category as the current article.

Fix: Add .eq('category', article.category) to the related articles Supabase query in articles/[slug]/page.tsx.

Low Hardcoded Sample Data in Production

Personnel data, campus locations, ticker headlines, and masthead stats are all from hardcoded sample data (lib/sample-data.ts), not fetched from the database. This means content updates require code changes and redeployment.

Fix: Migrate personnel, campus locations, and stats to Supabase tables with editorial management pages. Keep sample data as fallback only.

Low No Pagination on Articles Grid

All published articles render at once in the Articles Grid. As content grows, this will cause performance issues and long initial load times.

Fix: Add "Load More" button or infinite scroll with a limit of 9-12 articles per page.

Critical localStorage Used as Database for Critical Data

Author profiles, personnel edits, and review comments/decisions are all stored in localStorage instead of Supabase. This data is lost on browser clear, device switch, or incognito mode. Multiple editors cannot see each other's changes.

Fix: Migrate author profiles to profiles table, personnel edits to a personnel table, and review comments to a review_comments table in Supabase.

High Team Invite Feature is Broken

The Team Management page calls supabase.auth.admin.inviteUserByEmail() from the browser client. This requires the service role key and will always fail from the frontend.

Fix: Create a POST /api/team/invite server endpoint that uses createServiceRoleClient() to send invitations.

High Fix Buckets API Has No Auth

The /api/fix-buckets endpoint has NO authentication. Anyone can call it to create/modify storage bucket configurations. Should be removed or protected.

Fix: Either remove this endpoint entirely (it's a one-time setup utility) or add super_editor role verification.

Medium Review Workflow Uses Hardcoded Sample Data

The editorial review queue (/editorial/review) loads articles from hardcoded sampleArticles and stores review decisions in localStorage. It doesn't integrate with the real article database.

Fix: Query articles with status='review' from Supabase and store review decisions in a database table.

Critical Missing Auth Check on Article Edit Page

The /editorial/articles/[id]/edit page does NOT verify that the current user is the article author. Contributors can edit ANY article by navigating directly to the URL. The frontend hides the edit button, but there's no server-side enforcement.

Fix: Add authorization check: verify article.author_id === currentUser.id or user has editor/super_editor role before allowing edits.

7. Performance Optimization

Suggestions

Image Optimization

Gallery and article images use raw <img> tags. Switch to Next.js <Image> with automatic WebP/AVIF conversion, lazy loading, and responsive srcsets. Configure Supabase domain in next.config.ts:

Config: images: { remotePatterns: [{ hostname: 'psdwcxefdgncntvnitwf.supabase.co' }] }

Bundle Size: Lazy Load Heavy Components

Components like MagazineReader, CampusMap, and InaugurationCeremony are large but not visible initially. Use next/dynamic with ssr: false to lazy-load them.

Animation Performance

Multiple simultaneous CSS animations (hero-drift, curtain-wave, particles) can cause jank on lower-end devices. Use will-change: transform on animated elements and ensure animations use GPU-accelerated properties (transform, opacity only).

ISR: Increase Revalidation for Static Content

Homepage revalidation is set to 60 seconds. For mostly-static content like personnel and campus map, consider longer revalidation (300-600s) or on-demand revalidation via webhook.

Font Loading: Subset Fonts

Three Google Fonts are loaded (Playfair Display, Inter, JetBrains Mono). Consider subsetting to Latin-only if Devanagari isn't needed, reducing font file sizes by 50-70%.

Implementation Plan

10. Contributions Flow Improvements

Suggestions

Current Flow

User visits /submit → uploads file (JPEG/PNG/PDF/DOC/DOCX) → fills name, division, type → auto-detects category → submits → editors review in dashboard → approve creates article automatically.

Add Contribution Type Selector

Currently, the type (article/poem/photo/sketch) is auto-detected from file extension. Allow users to explicitly select their contribution type for better categorization. Add a dropdown: Article, Poem, Meme, Photo/Sketch, Painting, Other.

Add Category Selection for Contributors

Let contributors suggest a category (Campus, Culture, Sports, etc.) during submission. Editors can change it during review, but the suggestion helps with initial categorization.

Submission Status Tracking

Give contributors a submission ID and a way to check their submission status. Add a simple /submission/[id] page showing: Pending Review / Approved / Published / Rejected (with reason).

Rich Text Submission

For articles and poems, allow inline text submission (not just file upload). Add a simple TipTap editor in the submit form for direct text entry. This avoids the file conversion step during editorial review.

Email Notifications

If the contributor provides an email, send notifications when their submission is approved or published. Use Supabase Edge Functions or a simple email service integration.

Bulk Upload for Gallery Photos

Allow uploading multiple photos at once with shared metadata (category, event name). Useful for uploading entire event photo albums.
