Website Enhancement — Aesthetics, Animations & Features
Innovative ideas to elevate the visual impact, interactivity, and content richness of the GHORPAD digital magazine.
A.1 — VISUAL & AESTHETIC ENHANCEMENTS
1 Light Mode / Dual-Theme Toggle
Introduce a light mode alongside the current dark theme. A toggle in the navbar lets visitors switch between themes. The light theme could use warm cream (#FDF8EF) background with deep navy (#1A1A2E) text — evoking the feel of printed magazine paper. This improves readability in bright environments and makes the magazine feel more versatile. Use CSS custom properties (already in place) so every component adapts automatically.
AESTHETIC EFFORT: MEDIUM HIGH IMPACT
2 Parallax Scrolling Hero with Video Background
Replace or augment the particle canvas hero with a muted, looping drone footage video of the campus. The video auto-plays silently at low opacity (15-25%) behind the hero text, creating a cinematic first impression. Add a multi-layer parallax effect where the headline, sub-text, and CTA buttons move at different scroll speeds. This single change will dramatically elevate the landing experience.
AESTHETIC EFFORT: MEDIUM RECOMMENDED
3 Accent Colour Palette Expansion
The current gold + red palette is strong but can feel monochromatic. Introduce section-specific accent colours: teal for Sports, warm amber for Culture, cool blue for Tech, verdant green for Achievements. Each article category gets a subtle colour identity in card borders, category badges, and article page accent bars — while gold remains the dominant brand colour. This creates visual variety without breaking design coherence.
AESTHETIC EFFORT: LOW HIGH IMPACT
4 Animated Page Transitions Between Routes
Currently, navigating between pages (Home → Article → Media) is an instant browser load. Implement shared layout animations using Framer Motion's layoutId — when a user clicks an article card, the card image smoothly expands into the article's hero image. The article title morphs from card position to full-page headline. This creates a "magazine-like page turn" feel that is both delightful and premium.
AESTHETIC EFFORT: HIGH HIGH IMPACT
5 Scroll-Triggered Text Reveal Animations
Add character-by-character or word-by-word reveal animations on section headings as they enter the viewport. The heading text "The Leadership" in Who Is Who would cascade letter-by-letter from left to right with a gold shimmer trail. Use Framer Motion's staggerChildren on split text spans. Apply sparingly — only to section headers — so it feels premium rather than distracting.
AESTHETIC EFFORT: LOW RECOMMENDED
6 Microinteraction Polish — Cursor, Buttons & Cards
Add a custom cursor that transforms into a gold ring when hovering over interactive elements. Buttons get a subtle magnetic pull effect (cursor attracts towards button center). Cards get a 3D tilt effect on hover using CSS perspective and rotateX/rotateY mapped to mouse position. These micro-details signal craftsmanship and make the site feel alive.
AESTHETIC EFFORT: MEDIUM
A.2 — CONTENT & FEATURE INNOVATIONS
7 Video Embedding & Short Reels Section
The Media Vault currently has placeholder thumbnails. Enable actual video embedding — YouTube/Vimeo iframes for event coverage, or self-hosted short clips via Supabase Storage. Add a "Reels" sub-section with vertical short-form clips (15-60 seconds) of campus life, cultural events, and parade footage. These auto-play on scroll (muted) with a tap to unmute — mimicking the Instagram Reels format that the audience is already familiar with.
RECOMMENDED EFFORT: MEDIUM HIGH IMPACT
8 Interactive Photo Gallery with Masonry Layout
Replace the horizontal carousel in Media Vault with a full-screen masonry photo gallery (like Pinterest). Photos of varying aspect ratios tile elegantly. Clicking any photo opens a full-screen lightbox with swipe/arrow navigation, EXIF data (date, event), and download button. Organise galleries by event ("Tech Fest 2026", "Passing Out Parade", "Sports Day") with filterable tabs.
AESTHETIC EFFORT: MEDIUM HIGH IMPACT
9 Alumni Spotlight & "Where Are They Now" Section
Add a dedicated section showcasing notable alumni and their career trajectories after leaving the institute. Each spotlight includes a photo, current designation, a short testimonial quote, and their batch year. This builds institutional pride, provides aspirational content for current students, and strengthens alumni engagement. Can be implemented as a horizontally scrolling timeline or card carousel.
HIGH IMPACT EFFORT: MEDIUM
10 Interactive Campus Map
Embed an SVG-based interactive campus map where hovering over buildings highlights them with a gold glow and shows a tooltip with the building name and purpose. Clicking a building navigates to related content (e.g., clicking the Sports Complex scrolls to Sports articles). This is both visually striking and functionally useful for visitors unfamiliar with the campus layout.
AESTHETIC EFFORT: HIGH
11 Audio Articles — "Listen to This Story"
Add a play button to each article that reads the content aloud using the Web Speech API (browser-native, zero cost) or a premium TTS service. A persistent mini audio player bar at the bottom shows progress and allows pause/resume. This is excellent for accessibility and for readers who prefer audio consumption. The implementation is surprisingly simple — the browser's built-in SpeechSynthesis API requires no API keys.
RECOMMENDED EFFORT: LOW HIGH IMPACT
12 Commandant's Corner — Monthly Video Address
A prominent section on the homepage for a monthly recorded video message from the Commandant. The video is displayed in a cinematic widescreen player with the Commandant's name, date, and topic title overlaid. This humanises the leadership, creates a recurring reason to visit the site, and serves as an official communication channel. Archive past addresses in a scrollable timeline.
RECOMMENDED EFFORT: LOW
13 Countdown Timer for Upcoming Events
A subtle but eye-catching countdown widget in the hero or below the ticker showing the next major institute event ("Passing Out Parade in 12d 6h 34m"). Uses real-time JavaScript countdown. After the event passes, it automatically switches to the next one. Creates anticipation and a reason for repeat visits. Can be managed via a simple JSON config or the editorial dashboard.
EFFORT: LOW HIGH IMPACT
14 Achievement Showcase with Statistics Counter
An animated statistics bar showing key institute metrics — "1,200+ Officers Trained" · "48 Batches" · "12 Divisions" · "6 Sports Championships". Numbers count up from zero when they scroll into view using a simple requestAnimationFrame loop. This is visually impressive, requires minimal data, and immediately communicates the institute's scale and legacy.
AESTHETIC EFFORT: LOW RECOMMENDED
15 PDF Download — "Print Edition" of Full Magazine
Generate a downloadable PDF version of the complete magazine using a server-side PDF renderer (e.g., Puppeteer or @react-pdf/renderer). The PDF includes the cover, TOC, all articles, personnel directory, and media thumbnails — formatted for A4 printing. This bridges the gap between digital and print, allows offline reading, and serves as an official archive record.
EFFORT: HIGH HIGH IMPACT
16 Dark Mode Reading View for Articles
When reading a full article, offer a "Focus Mode" toggle that strips away navigation, sidebar, and footer — leaving only the article text in a clean, centred column with optimal line length (65-75 characters). Typography scales up slightly, line-height increases, and a warm reading light filter can be toggled on. This mimics the premium reading experience of apps like Medium and Instapaper.
AESTHETIC EFFORT: LOW

Editorial Workflow — Efficiency & Workload Reduction
Practical suggestions to streamline the editorial pipeline — from collecting contributions to publishing polished content with minimal friction.
B.1 — COLLECTING CONTRIBUTIONS FROM WRITERS
1 Public Submission Portal with Guided Form
Create a public-facing /submit page where anyone (student officers, staff, alumni) can submit an article without needing editorial login credentials. The form collects: author name, batch/division, email, article title, category, and content (rich text or file upload). Submissions land in a "Pending Review" queue visible only to editors. This eliminates the need to share login credentials or collect articles via WhatsApp/email.
WORKFLOW EFFORT: MEDIUM HIGHLY RECOMMENDED
2 WhatsApp/Email Submission Bot
Set up a dedicated WhatsApp Business number or email address (e.g., submit@ghorpad.in) where contributors can simply send their articles as messages or attachments. A Cloudflare Email Worker or WhatsApp webhook parses incoming messages and auto-creates draft entries in the editorial dashboard. This meets contributors where they already are — on their phones — rather than requiring them to visit a website.
WORKFLOW EFFORT: HIGH HIGH IMPACT
3 Google Docs Integration for Collaborative Writing
Allow contributors to write in Google Docs (a tool everyone knows). The editorial dashboard has an "Import from Google Docs" button that takes a shared doc URL and pulls the formatted content directly into TipTap. This preserves headings, bold/italic, images, and links. Contributors write in their comfort zone; editors import with one click. The existing DOCX upload feature already supports this workflow partially — Google Docs can export as .docx.
WORKFLOW EFFORT: LOW RECOMMENDED
4 Contribution Deadline Tracker & Reminders
Add a deadlines panel to the editorial dashboard showing upcoming submission deadlines for each issue. Editors set a deadline date; the system shows a countdown and can send automated email reminders to registered contributors 7 days, 3 days, and 1 day before the deadline. This reduces the "chasing contributors" burden that plagues every editorial team.
WORKFLOW EFFORT: MEDIUM
B.2 — AUTHOR PROFILES & BIOGRAPHICAL DATA
5 Self-Service Author Profile Page
When a contributor first logs in (or submits), prompt them to fill out a one-time author profile: display name, short bio (150 chars), full bio, photo upload, batch/division, and social links. This profile auto-attaches to every article they write. Editors never have to manually collect or write author bios again. The author can update their own profile at any time without editor involvement.
WORKFLOW EFFORT: MEDIUM HIGHLY RECOMMENDED
6 AI-Assisted Bio Generator
For personnel profiles that need bios, offer a "Generate Bio" button in the editorial personnel page that creates a professional 2-3 sentence bio from the available data (name, rank, designation, regiment, achievements). Use a simple template engine or an AI API call. This saves editors from writing 200+ individual bios from scratch. The generated text is always editable before saving.
WORKFLOW EFFORT: LOW HIGH IMPACT
B.3 — PHOTOGRAPH COLLECTION & MANAGEMENT
7 Bulk Photo Upload via QR Code Link
Generate a unique QR code for each event or personnel batch. Officers scan the QR with their phone, which opens a simple upload page where they can select multiple photos from their gallery. Photos are tagged with the event name and uploader's identity automatically. Display this QR code at events or circulate via WhatsApp. This is vastly more efficient than collecting photos through messaging groups.
WORKFLOW EFFORT: MEDIUM HIGHLY RECOMMENDED
8 Auto-Resize & Optimise on Upload
Phone cameras produce 5-15MB images that slow down the website. Implement automatic image processing on upload: resize to max 1920px width, convert to WebP format (60-70% smaller than JPEG), and generate thumbnail variants (400px, 800px). Use Sharp or a Vercel Edge Function. Store the original as a backup. This happens invisibly — the uploader sees no extra steps.
WORKFLOW EFFORT: MEDIUM HIGH IMPACT
9 Personnel Photo Collection Campaign Template
Create a reusable template page (e.g., /photos/staff-2026) with a simple interface: each person's name is listed with a "Upload Your Photo" button next to it. Officers click their name, upload a passport-size photo, and the system auto-crops and links it to their personnel record. Include visual guidelines (passport-size, formal dress, plain background) directly on the page. This eliminates the chaos of collecting 200+ photos via WhatsApp.
WORKFLOW EFFORT: MEDIUM HIGHLY RECOMMENDED
B.4 — QUALITY ASSURANCE & ACCURACY
10 Pre-Publish Checklist
Before an article can be published, enforce a mandatory checklist: ✓ Title under 80 characters · ✓ Excerpt filled · ✓ Cover image set · ✓ Category assigned · ✓ Spell-check run · ✓ Author bio attached · ✓ At least one tag. If any item fails, the "Publish" button is disabled with a clear explanation. This catches common oversights and ensures consistent quality across every article without manual review.
WORKFLOW EFFORT: LOW HIGHLY RECOMMENDED
11 Built-In Spell & Grammar Check
Integrate a lightweight grammar checker into the TipTap editor. Options range from free (browser's built-in spellcheck is already active) to premium (LanguageTool API — free tier: 20 requests/minute, catches grammar, style, and punctuation errors). Underline issues in red directly in the editor. This catches errors before they reach the review stage, reducing back-and-forth between editors and contributors.
WORKFLOW EFFORT: LOW
12 Two-Stage Review Workflow with Comments
Extend the article status from "Draft → Published" to "Draft → In Review → Approved → Published". When an article is submitted for review, editors can leave inline comments (highlight text + add note) without modifying the original. The contributor sees the comments, makes revisions, and resubmits. This creates an auditable review trail and avoids editors rewriting contributor's work silently.
WORKFLOW EFFORT: HIGH HIGH IMPACT
13 Duplicate & Plagiarism Detection
Before publishing, run a quick similarity check against previously published articles to catch duplicate submissions or excessive copying. A simple approach: hash paragraph-level n-grams and compare against the existing article database. Flag articles with >30% overlap. This protects the magazine's credibility and ensures content originality.
WORKFLOW EFFORT: MEDIUM
B.5 — DASHBOARD & AUTOMATION
14 Analytics Dashboard — Readership Metrics
Add Vercel Web Analytics (included in Pro plan) or Plausible Analytics (privacy-focused) to track: page views per article, most-read articles, visitor geography, device breakdown, and referral sources. Display these as charts in the editorial dashboard. Knowing which articles resonate helps the editorial team make data-driven decisions about future content focus areas.
WORKFLOW EFFORT: LOW RECOMMENDED
15 Scheduled Publishing
Allow editors to schedule an article for future publication — set a date and time, and the article goes live automatically. This enables the team to batch-prepare content in advance and have it release at optimal times, even during weekends or holidays. Implementation: a scheduled_at timestamp field + a cron job or Supabase Edge Function that flips status to "published" when the time arrives.
WORKFLOW EFFORT: MEDIUM HIGH IMPACT
16 Article Templates for Common Formats
Pre-fill new articles with reusable templates: "Event Coverage" (date, venue, attendees, highlights, quotes), "Interview" (introduction, Q&A format, closing), "Opinion Piece" (thesis, arguments, conclusion), "Sports Report" (match details, scores, standout performers). Contributors select a template and fill in the blanks rather than staring at a blank editor. This dramatically improves content consistency and reduces writing time.
WORKFLOW EFFORT: LOW RECOMMENDED

Security Hardening — Threat Mitigation & Best Practices
Layered security measures to protect the GHORPAD website from common attack vectors, data breaches, and operational risks.
C.1 — CURRENT SECURITY POSTURE ASSESSMENT

What's already in place:

✓ Middleware-protected editorial routes (auth check on every /editorial/* request)

✓ Role-based access control (super_editor / editor / contributor)

✓ Supabase Auth with JWT session management

✓ HTTPS enforced via Vercel (Let's Encrypt auto-provisioned)

✓ Environment variables for secrets (.env.local, not committed to git)

What needs attention:

✗ No Row Level Security (RLS) policies defined in Supabase yet

✗ No rate limiting on editorial API routes

✗ No Content Security Policy (CSP) headers

✗ No input sanitisation on article content (XSS risk via TipTap)

✗ Demo mode credentials are hardcoded and publicly visible

✗ No file upload validation beyond file extension

✗ No audit logging of editorial actions

C.2 — THREAT MODEL & MITIGATIONS
Threat Severity Current Status Recommended Mitigation
Unauthenticated API Access
Direct Supabase queries bypassing frontend		 CRITICAL No RLS policies Enable RLS on all tables. Write policies using auth.uid(). Test with Supabase impersonation.
Cross-Site Scripting (XSS)
Malicious HTML/JS in article content		 HIGH TipTap allows HTML Sanitise all TipTap output with DOMPurify before rendering. Strip <script>, onerror, onclick attributes.
Malicious File Upload
Executable files disguised as images		 HIGH Extension check only Validate MIME type on server side. Check magic bytes. Limit file size (5MB). Scan with ClamAV if budget allows.
Brute Force Login
Automated password guessing		 HIGH No rate limiting Implement rate limiting (5 attempts / 15 min). Use Supabase's built-in rate limits. Add CAPTCHA after 3 failures.
Session Hijacking
Stolen JWT tokens		 MEDIUM HttpOnly cookies via Supabase SSR Set short JWT expiry (1 hour). Use refresh tokens. Bind sessions to IP/User-Agent fingerprint.
CSRF Attacks
Forged cross-site requests		 MEDIUM No explicit CSRF protection Use SameSite=Strict cookies. Add CSRF tokens to state-changing forms. Verify Origin header in API routes.
Clickjacking
Site embedded in malicious iframes		 LOW No X-Frame-Options Add X-Frame-Options: DENY and Content-Security-Policy: frame-ancestors 'none' headers.
Data Exfiltration
Personnel PII (emails, phone numbers)		 HIGH Stored in plaintext Encrypt PII at rest. Restrict API access to PII fields. Don't expose contact details to unauthenticated users.
C.3 — PRIORITY SECURITY ACTIONS (ORDERED)
1 Enable Row Level Security on All Supabase Tables
This is the single most important security action. Without RLS, anyone with your Supabase anon key (which is exposed in the browser) can read, modify, or delete all data in your database. Enable RLS on every table and write policies: public users can only SELECT published articles; only authenticated editors can INSERT/UPDATE; only super_editors can DELETE. Test every policy using Supabase Studio's impersonation feature.
SECURITY EFFORT: MEDIUM CRITICAL PRIORITY
2 Content Security Policy (CSP) Headers
Add strict CSP headers via next.config.ts security headers or Vercel configuration. Recommended baseline: default-src 'self'; script-src 'self' 'unsafe-eval'; style-src 'self' 'unsafe-inline' fonts.googleapis.com; img-src 'self' data: blob: *.supabase.co; font-src fonts.gstatic.com; connect-src 'self' *.supabase.co;. This prevents injected scripts from loading external resources and is the primary XSS mitigation layer.
SECURITY EFFORT: LOW HIGH PRIORITY
3 Input Sanitisation with DOMPurify
Install dompurify and sanitise all TipTap HTML content before rendering on the public site. Never trust user-generated HTML. DOMPurify strips dangerous tags (<script>, <iframe>) and attributes (onerror, onload, onclick) while preserving safe formatting. Apply sanitisation both on save (defence in depth) and on render (display safety).
SECURITY EFFORT: LOW HIGH PRIORITY
4 Secure File Upload Pipeline
Harden the file upload flow: (a) Validate MIME type using file magic bytes, not just extension. (b) Enforce max file sizes (images: 5MB, documents: 10MB). (c) Rename files to random UUIDs (prevents path traversal). (d) Store in a private Supabase bucket; serve via signed URLs with expiry. (e) Strip EXIF metadata from uploaded images (may contain GPS coordinates — a privacy concern for military personnel).
SECURITY EFFORT: MEDIUM HIGH PRIORITY
5 Rate Limiting & Brute Force Protection
Add rate limiting to: (a) Login endpoint — max 5 attempts per 15 minutes per IP. (b) API routes — max 100 requests per minute per user. (c) File uploads — max 20 uploads per hour. Implement using Vercel Edge Middleware with a simple in-memory counter or Upstash Redis for distributed rate limiting. After 3 failed logins, display a CAPTCHA (hCaptcha is free and privacy-friendly).
SECURITY EFFORT: MEDIUM HIGH PRIORITY
6 Security Headers Configuration
Add these headers in next.config.ts:
  • X-Frame-Options: DENY — prevents clickjacking
  • X-Content-Type-Options: nosniff — prevents MIME-type sniffing
  • Referrer-Policy: strict-origin-when-cross-origin — limits referrer data leakage
  • Permissions-Policy: camera=(), microphone=(), geolocation=() — disables unnecessary browser APIs
  • Strict-Transport-Security: max-age=63072000; includeSubDomains — enforces HTTPS for 2 years
SECURITY EFFORT: LOW RECOMMENDED
7 PII Protection for Personnel Data
Personnel records contain sensitive PII (phone numbers, email addresses, family details). (a) Never expose PII to unauthenticated users via API — the public site should show names, ranks, and designations only. (b) Contact details visible only when a user is logged in, or behind a "click to reveal" that logs access. (c) Strip EXIF/GPS data from personnel photos. (d) Consider encrypting PII columns at the database level using Supabase Vault (pgcrypto).
SECURITY EFFORT: MEDIUM CRITICAL — MILITARY CONTEXT
8 Audit Logging
Log every significant editorial action: who did what at when. Track: article publishes/unpublishes, personnel edits, media uploads/deletions, login attempts (success and failure), and role changes. Store in a separate audit_logs table that editors cannot delete. This creates accountability and enables forensic analysis if something goes wrong.
SECURITY EFFORT: MEDIUM RECOMMENDED
9 Disable Demo Mode in Production
The demo mode (hardcoded credentials: editor@ghorpad.in / demo1234) must be completely disabled before deployment to production. Ensure the NEXT_PUBLIC_DEMO_MODE environment variable is set to false or removed entirely in the Vercel production environment. Add a build-time check that fails the build if demo mode is enabled in production.
SECURITY EFFORT: LOW CRITICAL PRIORITY
10 Backup & Disaster Recovery Strategy
(a) Enable Supabase Point-in-Time Recovery (PITR) on the Pro plan — allows restoring the database to any second within the last 7 days. (b) Set up weekly automated exports of the database via pg_dump to a separate storage location. (c) Keep the Git repository on a private GitHub/GitLab with branch protection (no force-push to main). (d) Document the recovery procedure so any team member can restore the site if needed.
SECURITY EFFORT: MEDIUM RECOMMENDED
C.4 — PRE-LAUNCH SECURITY CHECKLIST

Complete these items before the website goes live:

  • Supabase RLS enabled on articles, profiles, media, and personnel tables
  • RLS policies tested using Supabase Studio impersonation for each role
  • Demo mode disabled — environment variable removed or set to false
  • Service role key never used in frontend code (grep codebase to verify)
  • Security headers configured in next.config.ts
  • CSP header deployed and tested (check browser console for violations)
  • DOMPurify sanitising all rendered article HTML
  • File upload validation — MIME check, size limit, EXIF stripping
  • HTTPS enforced with HSTS header (automatic on Vercel)
  • Environment variables — no secrets in client-side code (grep for SUPABASE_SERVICE_ROLE)
  • Git repository is private with branch protection on main
  • Supabase storage buckets set to private (serve via signed URLs)
  • Login rate limiting active (test with 6+ rapid incorrect logins)
  • Personnel PII not exposed to unauthenticated API requests
  • Error pages do not leak stack traces or internal paths in production
  • Dependencies audited — run npm audit and fix critical vulnerabilities
  • Backup strategy documented and tested with a trial restore
