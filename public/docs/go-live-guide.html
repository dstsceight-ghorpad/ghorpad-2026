// 03 — FRONTEND HOSTING

Vercel — Is It Required?

What Vercel does for this project

Vercel hosts the Next.js application. It runs the Node.js server that renders pages, handles the authentication middleware, and serves static assets (CSS, JavaScript, images) to visitors.

Why Vercel is recommended

1. Zero-config deployment. Vercel is built by the same company that created Next.js. You connect your GitHub repo, and it automatically detects the framework, installs dependencies, builds, and deploys. No server setup, no Docker, no DevOps knowledge needed.

2. This site needs a Node.js server. GHORPAD uses middleware (for auth protection on editorial routes) and dynamic server-rendered pages (article pages with [slug]). This means it cannot be exported as a plain static HTML site. It needs a server running Node.js.

3. Free tier is generous. 100 GB bandwidth/month, automatic HTTPS, global CDN, preview deployments on every git push — all free.

Can you deploy WITHOUT Vercel?

Yes, but it takes significantly more effort. Vercel is not a hard technical requirement. You can deploy on any platform that runs Node.js. Here are the alternatives:
Alternative Free Tier? Effort Tradeoffs
Netlify Yes (limited) Medium Next.js support is second-class. Middleware and SSR may behave differently. Some features may not work correctly.
Railway.app $5 credit/mo Medium Good Node.js support, but the free tier runs out quickly with continuous uptime. Site may sleep after inactivity.
Render.com Yes (limited) Medium Free web services spin down after 15 min of inactivity. First visit after sleep takes 30–60 seconds. Not ideal for a live site.
Your own VPS
(DigitalOcean, AWS EC2)		 No ($5–12/mo) High Full control, but you must set up Node.js, Nginx reverse proxy, SSL certificates, process manager (PM2), and handle updates manually.

Recommendation: Use Vercel free tier. It is the path of least resistance — one-click deploy, zero maintenance, zero cost, and guaranteed compatibility since Vercel created Next.js.

// 04 — BACKEND / DATABASE

Supabase — Is It Required?

What Supabase does for this project

Supabase provides three services that the editorial system uses:

🔒

1. Authentication (Login System)

Editors log in with email/password. Supabase handles password hashing, session tokens, and cookie management. Three roles are supported: super_editor, editor, contributor.

🗃

2. PostgreSQL Database (3 tables)

TablePurposeExpected Rows
profilesUser accounts and roles3–10 editors
articlesArticle content, drafts, metadata20–100 articles
mediaUploaded file metadata (URL, size, type)50–200 records

Row-Level Security (RLS) ensures editors can only edit their own drafts, while super_editors can manage everything.

📎

3. File Storage (2 buckets)

BucketPurposeExpected Usage
article-coversCover images for articles20–100 images (~50–200 MB)
mediaGeneral photos, videos uploaded by editors50–500 files (~200–800 MB)

Can you deploy WITHOUT Supabase?

Partially. The website has a built-in "demo mode" that activates when Supabase is not configured. Here is exactly what works and what does not:
Feature Without Supabase With Supabase
Public homepage (all sections) ✓ Works (sample data) ✓ Works (live data)
Photo Gallery, Alumni, Campus Map ✓ Works (sample data) ✓ Works
Magazine reader (flip-book) ✓ Works (sample data) ✓ Works
Editorial login ⚠ Demo only (hardcoded credentials) ✓ Real email/password
Write & save articles ✗ Not saved (lost on refresh) ✓ Saved to database
Upload photos / media ✗ Not stored ✓ Stored in cloud
Publish articles to public site ✗ Cannot publish ✓ Publish & goes live
Review workflow & team management ⚠ Demo only (localStorage) ✓ Fully functional
Public submission portal ⚠ Form works but data not saved ✓ Submissions saved
Bottom line: Without Supabase, the site is a read-only showcase displaying sample/placeholder data. You cannot write, publish, or manage real content. For a functioning magazine, Supabase is required.

Could you use a different backend instead of Supabase?

Alternative Free? Effort to switch Notes

Neon (PostgreSQL only) Yes High Free PostgreSQL, but no auth or file storage. You would need to add a separate auth solution and file hosting.
Recommendation: Use Supabase free tier. The entire codebase is built around it. Switching would require weeks of rewriting. The free plan gives you everything you need.

ℹ TIP: KEEP FILE STORAGE LEAN

The tightest limit is Supabase file storage at 1 GB. To stay within it: compress images before uploading (the site has built-in auto-resize), avoid uploading raw camera photos (10–20 MB each), and prefer JPEG/WebP over PNG for photographs. The built-in image optimizer already resizes cover images to reasonable dimensions.

// 08 — DEPLOYMENT PROCEDURE

Step-by-Step Go-Live Procedure

Follow these steps in order. Estimated total time: 30–45 minutes.

1

Push Code to GitHub

If not already done, push the project to a GitHub repository (public or private — both work).

cd ghorpad-2026
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/ghorpad-2026.git
git push -u origin main

Time: 5 minutes

2

Create a Supabase Project

1. Go to supabase.com and sign up (free).
2. Click New Project.
3. Name it ghorpad-2026.
4. Choose a strong database password (save it somewhere safe).
5. Select the region closest to your users (e.g., South Asia (Mumbai) if available, else Southeast Asia (Singapore)).
6. Click Create new project and wait ~2 minutes for it to initialize.

Time: 5 minutes

3

Set Up the Database Schema

1. In the Supabase dashboard, go to SQL Editor.
2. Open the file supabase/schema.sql from your project.
3. Paste the entire contents into the SQL editor.
4. Click Run.

This creates the 3 tables (profiles, articles, media), sets up Row-Level Security policies, creates the storage buckets, and adds the trigger for auto-creating user profiles.

-- You should see output like:
-- SUCCESS: Created table "profiles"
-- SUCCESS: Created table "articles"
-- SUCCESS: Created table "media"
-- SUCCESS: Created 2 storage buckets

Time: 5 minutes

4

Copy Supabase Credentials

1. In the Supabase dashboard, go to Settings → API.
2. Copy three values:

SettingWhere to Find
Project URLUnder "Project URL" — looks like https://xxxx.supabase.co
anon public keyUnder "Project API keys" → anon public
service_role keyUnder "Project API keys" → service_role (click "Reveal")

Time: 2 minutes

5

Create Your First Editor Account

1. In the Supabase dashboard, go to Authentication → Users.
2. Click Add user → Create new user.
3. Enter the email and password for the super editor (chief editor).
4. The trigger will auto-create a profile with role = contributor.
5. Go to Table Editor → profiles, find the new row, and change the role to super_editor.
6. Optionally set the full_name field.

Time: 3 minutes

6

Deploy to Vercel

1. Go to vercel.com and sign up with your GitHub account.
2. Click Add New → Project.
3. Import your ghorpad-2026 GitHub repository.
4. Before clicking Deploy, expand Environment Variables and add these three:

NEXT_PUBLIC_SUPABASE_URL       = https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY  = eyJhb...your_anon_key
SUPABASE_SERVICE_ROLE_KEY      = eyJhb...your_service_role_key

5. Click Deploy.
6. Wait ~60 seconds for the build to complete.
7. Your site is now live at ghorpad-2026.vercel.app!

Time: 5 minutes

7

Add Custom Domain (Optional)

1. In the Vercel dashboard, go to your project → Settings → Domains.
2. Type your domain (e.g., ghorpad.in) and click Add.
3. Vercel will show you DNS records to add at your domain registrar.
4. Log into your registrar (GoDaddy, Namecheap, etc.) and add those DNS records.
5. Wait 5–30 minutes for DNS propagation.
6. Vercel automatically provisions an SSL certificate.

Time: 10–30 minutes (mostly waiting for DNS)

8

Verify Everything Works

Run through this checklist:

CheckHow to Test
Public site loadsVisit the URL — homepage should show all sections
Editorial login worksGo to /editorial/login, log in with the editor account
Article creation worksCreate a draft article, save it, verify it persists after refresh
Media upload worksUpload an image in the Media Library
Publishing worksPublish an article, verify it appears on the public site
HTTPS worksCheck that the padlock icon appears in the browser address bar

Time: 5 minutes

// 09 — CONFIGURATION

Environment Variables Reference

Variable Where to Set Public? Purpose
NEXT_PUBLIC_SUPABASE_URL Vercel Dashboard → Environment Variables Yes (safe in browser) Tells the website where your Supabase project lives
NEXT_PUBLIC_SUPABASE_ANON_KEY Vercel Dashboard → Environment Variables Yes (safe in browser, restricted by RLS) Allows the browser to talk to Supabase. Row-Level Security ensures users can only access what they are allowed to.
SUPABASE_SERVICE_ROLE_KEY Vercel Dashboard → Environment Variables No — SECRET Server-side only key for admin operations. Never expose this in client-side code. Vercel keeps it secure.
⚠ SECURITY NOTE

The SUPABASE_SERVICE_ROLE_KEY bypasses all Row-Level Security. Never commit it to git, never put it in .env.local that gets pushed, and never include it in client-side code. In Vercel, it is automatically kept on the server side.

// 11 — FAQ

Frequently Asked Questions

Q: What happens if the site goes viral and gets 100,000 visitors?

Vercel will handle the traffic without issues (100 GB bandwidth is a lot). If Supabase bandwidth exceeds 5 GB, the database calls may be throttled. In practice, public pages load fast from Vercel's CDN and make minimal database calls — most traffic is served from cache.

Q: Can I deploy this on the college's own server?

Yes, if the college server runs Node.js 18+ and has a public IP. You would need to: install Node.js, clone the repo, run npm install && npm run build && npm start, set up a reverse proxy (Nginx), and configure SSL. You would still need Supabase (cloud) for the database, or set up your own PostgreSQL server.

Q: Can I add Google Analytics?

Yes. Add the GA tracking script to app/layout.tsx. It is free and does not affect hosting costs. Alternatively, use Plausible or Umami for privacy-friendly analytics (both have free self-hosted options).

Q: How do I update the site after it is live?

Just push code changes to the GitHub main branch. Vercel automatically detects the push, rebuilds, and deploys within ~60 seconds. Zero downtime.

Q: What if Supabase or Vercel discontinues free plans?

Both companies have maintained free tiers for years and are well-funded. If they ever remove free plans, you would need to migrate. The code is portable — Next.js can run on any Node.js host, and the database is standard PostgreSQL that can be exported and imported anywhere.

Q: I am only getting the demo/sample data on the public site. Why?

The public homepage currently always displays sample data from lib/sample-data.ts, regardless of Supabase configuration. This is by design during development. To show live database articles instead, the components need to be updated to fetch from Supabase when not in demo mode. The editorial system already uses real Supabase data.

Q: Who should be the Vercel account holder?

Ideally, an institutional account or a senior team member who will be available long-term. The Vercel account holder controls deployments and environment variables. Content management (writing and publishing articles) is handled through the editorial portal and does not need Vercel access.
